FILES TO UPDATE FOR NEW SUPABASE PROJECT
=========================================

ONLY .ENV FILE NEEDS TO BE UPDATED
-----------------------------------

1. .env file
   - Update database credentials:
     * DB_CONNECTION=pgsql
     * DB_HOST=new-project-host.supabase.co
     * DB_PORT=5432
     * DB_DATABASE=postgres
     * DB_USERNAME=postgres
     * DB_PASSWORD=new-password
     * DB_SSLMODE=require
   - OR use DB_URL with connection string

NO CODE FILES NEED CHANGES
---------------------------

All code files use environment variables from .env:
- config/database.php - reads from env() ✅
- app/Http/Controllers - no hardcoded values ✅
- routes/web.php - no database references ✅
- app/Models - no hardcoded values ✅

DOCUMENTATION FILES (OPTIONAL)
------------------------------

These files mention the old project reference but don't affect functionality:
- *.md files (documentation only)
- *.txt files (guides only)

You can update these later if you want, but they're not required for the app to work.

WHAT TO DO:
-----------

1. Create new Supabase project
2. Get connection string from Supabase dashboard
3. Update .env file with new credentials
4. Run: php artisan config:clear
5. Run: php artisan migrate
6. Done! ✅

NO OTHER FILES NEED TO BE CHANGED!
