SUPABASE CONNECTION GUIDE - HOW TO GET YOUR CONNECTION STRING
================================================================

For Supabase, the username is ALWAYS "postgres" by default.

WHERE TO FIND CONNECTION DETAILS IN SUPABASE:

1. Go to: https://supabase.com/dashboard/project/qmonmawebxzlbithvkli

2. Click "Settings" (gear icon in left sidebar)

3. Click "Database" (under Project Settings)

4. Scroll down to "Connection string" section

5. You'll see different connection modes:
   - Direct connection
   - Connection pooling (Transaction mode)
   - Connection pooling (Session mode)

6. Click the "URI" tab or "Connection parameters" tab

7. You'll see either:
   - A full connection string: postgresql://postgres:[PASSWORD]@[HOST]:5432/postgres
   - OR separate parameters showing Host, Port, Database, User, Password

FOR YOUR .ENV FILE:

Option 1: Use Connection String (EASIEST)
------------------------------------------
Copy the ENTIRE connection string from Supabase and use it in .env:

DB_URL=postgresql://postgres:YOUR_PASSWORD@HOST:5432/postgres?sslmode=require

Replace YOUR_PASSWORD with your actual database password.


Option 2: Use Separate Parameters
----------------------------------
From Supabase dashboard, extract:

DB_CONNECTION=pgsql
DB_HOST=[HOST from Supabase]
DB_PORT=5432 (or 6543 for pooling)
DB_DATABASE=postgres
DB_USERNAME=postgres  <-- ALWAYS "postgres" for Supabase
DB_PASSWORD=[PASSWORD from Supabase]
DB_SSLMODE=require

IMPORTANT:
- Username is ALWAYS "postgres" (unless using connection pooling with project ref)
- Password is the database password (you set this when creating the project)
- If you forgot password: Settings → Database → Reset database password
- Host is shown in Connection string or Connection parameters

3. Copy the ENTIRE connection string
4. Use it in your .env as DB_URL=...
5. OR extract the HOST and PASSWORD from it
